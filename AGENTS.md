# Agent Instructions

## Changelog

Every change to this repository must be reflected in the `## Changelog` section of `README.md`.

- Add a new `### <version> — <date>` entry at the top of the Changelog section for each release or meaningful batch of changes.
- Summarise what changed and why — not just what files were touched.
- Keep entries in reverse-chronological order (newest first).
- If a change makes any existing README prose stale (e.g. the Security section), update that prose in the same commit.
